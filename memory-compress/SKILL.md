# Memory Compress

自动化记忆压缩，保持记忆系统高效运转。

## 核心原则

- **保留**：决策、教训、关键事件、人物偏好、项目状态
- **压缩**：重复信息合并、流水账提炼为摘要
- **丢弃**：纯调试日志、已解决的临时问题、过期的一次性信息
- **归档**：超过 30 天的日志移入 memory/archive/

## 压缩流程

### 1. 扫描

```bash
ls -la ~/. openclaw/workspace/memory/*.md
```

统计文件数量和总大小。

### 2. 分类处理

**日志文件 (memory/YYYY-MM-DD.md)**

对每个超过 7 天的日志：
- 提取关键信息（决策、教训、重要事件）
- 检查是否已被 MEMORY.md 收录
- 未收录的关键信息 → 写入 MEMORY.md
- 超过 30 天 → 移入 memory/archive/YYYY-MM/

**MEMORY.md**

- 删除已过时的信息（已完成的项目、已失效的 token 等）
- 合并重复条目
- 确保结构清晰，按主题分类

**专项文件 (failures.md, capabilities.md 等)**

- 保留所有条目（这些本身就是压缩后的知识）
- 检查是否有过时条目需要更新

### 3. 生成报告

输出压缩结果：
- 处理了多少文件
- 压缩前后大小对比
- 归档了哪些文件
- MEMORY.md 新增/删除了什么

## 触发方式

1. **自动触发**：每周审计时顺带执行
2. **手动触发**：用户说"压缩记忆"、"整理记忆"
3. **阈值触发**：memory/ 下文件超过 50 个或总大小超过 500KB

## 归档结构

```
memory/
├── archive/
│   ├── 2026-02/
│   │   ├── 2026-02-15.md
│   │   └── ...
│   └── 2026-03/
├── 2026-03-15.md  (近期保留)
├── 2026-03-16.md
├── 2026-03-17.md
├── YYYY-MM-DD.md
├── failures.md
├── capabilities.md
├── agent-stats.json
└── index.md
```
