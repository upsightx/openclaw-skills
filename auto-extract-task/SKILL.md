# Auto Extract Task

从对话中自动识别和提取任务，避免遗漏。

## 识别规则

### 强信号（直接创建任务）
- 明确指令："帮我创建一个任务"、"加到待办"、"记下来"
- 带截止时间的行动项："明天之前把XX做完"、"周五前提交"
- 承诺类："我答应了XX要做YY"

### 中信号（确认后创建）
- 隐含任务："这个问题需要解决"、"回头看看XX"
- 讨论中的结论："那就这么定了，先做XX"
- 提到他人期望："XX说让我们做YY"

### 弱信号（记录但不创建）
- 模糊意向："有空的话看看"、"以后可以考虑"
- 纯讨论无结论的话题

## 提取字段

从对话中提取以下信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| summary | ✅ | 任务标题（一句话） |
| description | ❌ | 任务详情（对话上下文） |
| due | ❌ | 截止时间（从对话中推断） |
| priority | ❌ | 优先级（urgent/high/medium/low） |
| assignee | ❌ | 负责人（默认主人） |

## 创建方式

### 飞书任务（默认）
使用 `feishu_task_task` 工具创建：
```
action: create
summary: [任务标题]
description: [任务详情]
due: { timestamp: [ISO 8601 时间] }
members: [{ id: "<user_open_id>", role: "assignee" }]
current_user_id: "<user_open_id>"
```

### 本地记录（备选）
如果飞书任务创建失败，写入 `memory/pending-tasks.md`：
```markdown
- [ ] [任务标题] | 截止: [时间] | 优先级: [级别]
  - 来源: [对话日期和上下文]
```

## 执行时机

1. **实时提取**：对话中识别到强信号时，立即提取并确认
2. **对话结束回顾**：长对话结束时，回顾是否有遗漏的任务
3. **每日回顾**：heartbeat 时检查 memory/pending-tasks.md 中未处理的项

## 确认策略

- 强信号：直接创建，创建后告知主人
- 中信号：先问主人"要不要创建一个任务：[标题]？"
- 弱信号：只记录到 daily notes，不打扰

## 注意事项

- 不要过度提取，避免把每句话都变成任务
- 主人说"不用了"、"算了"时，立即停止
- 重复的任务不要重复创建
- 优先级默认 medium，除非对话中有明确的紧急程度
